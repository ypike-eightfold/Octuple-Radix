# Design Tokens - Octuple V2.5

The Octuple V2.5 design system uses a comprehensive set of design tokens to ensure consistency across all components. These tokens define colors, typography, spacing, and more.

## Usage in Code

### Direct Color Values

```css
.my-component {
  /* Primary brand color */
  background-color: #1999ac;
  
  /* Text colors */
  color: #1a212e; /* Primary text */
  
  /* Backgrounds */
  background: #f6f7f8; /* Light grey */
  border: 1px solid #e3e8ee;
}
```

### Spacing & Border Radius

```css
.my-button {
  padding: 12px 16px; /* $space-s $space-m */
  border-radius: 8px; /* $corner-radius-m */
  gap: 8px; /* $space-xs */
}
```

### Typography

```tsx
<div style={{ 
  fontFamily: "'Gilroy', 'Poppins', sans-serif",
  fontSize: '16px',
  fontWeight: 600,
  lineHeight: '24px',
  color: '#1a212e'
}}>
  Heading Text
</div>
```

### Component Integration

```tsx
import { ButtonV2, ToastV2, InputV2 } from '@eightfold.ai/octuple-radix';

// All V2.5 components automatically use these design tokens
<ButtonV2 variant="primary" size="medium">
  Click Me
</ButtonV2>
```

| **Border Radius** | 4px - 24px (5 values) | Component roundness |

| **Icons** | Material Symbols + Brand | System icons + product icons |
| **Shadows** | 4 elevation levels | Cards, dropdowns, modals |

---

## Design Principles

1. **Consistency:** Use tokens consistently across all components
2. **Accessibility:** Ensure sufficient color contrast (WCAG AA minimum)
3. **Flexibility:** Tokens support both light and dark themes
4. **Scalability:** Component sizes scale proportionally
5. **Semantic:** Color names reflect their purpose (success, error, warning)

For component-specific implementations, refer to individual component documentation in the V2.5 section.
